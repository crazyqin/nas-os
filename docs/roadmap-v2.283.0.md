# NAS-OS v2.283.0 版本路线图

## 版本概述

**版本号**: v2.283.0
**发布类型**: 安全增强版
**计划发布日期**: 2026-04-15
**前置版本**: v2.282.0

### 版本定位

v2.283.0 聚焦于**企业级安全能力**和**高性能存储扩展**：
- 勒索软件检测与防护（对标TrueNAS 26）
- NVMe-oF支持（高性能存储网络）
- 发布节奏调整（年度大版本模式）

---

## 核心功能规划

### 🔒 P0 - 勒索软件检测与防护

**背景**: TrueNAS 26推出勒索软件检测功能，企业安全需求日益增长

**目标**: 实时检测并阻止勒索软件攻击，零数据丢失

| 子功能 | 描述 | 预计工时 | 负责 |
|--------|------|----------|------|
| 文件事件监控器 | 实时捕获文件系统事件，fsnotify+inotify | 8h | 兵部 |
| 行为分析器 | 多规则检测引擎，识别勒索软件特征行为 | 12h | 兵部 |
| 熵值分析器 | 检测文件内容熵值变化，识别加密行为 | 6h | 兵部 |
| 威胁引擎 | 多因子评分，决策是否触发防护 | 8h | 兵部 |
| 隔离服务 | 终止恶意进程、锁定文件、阻断网络 | 6h | 工部 |
| 恢复服务 | 实时备份、一键恢复被加密文件 | 8h | 兵部 |
| 告警服务 | 多渠道通知（邮件/Webhook/推送） | 4h | 工部 |
| WebUI界面 | 威胁面板、规则配置、恢复操作 | 6h | 兵部 |

**设计文档**: `docs/ransomware-detection-design.md`（已完成）

**里程碑**:
- M1 (04-01): 核心检测引擎完成
- M2 (04-08): 隔离与恢复服务完成
- M3 (04-12): WebUI与集成测试完成

---

### ⚡ P0 - NVMe over Fabrics (NVMe-oF) 支持

**背景**: TrueNAS 25.04引入NVMe-oF，存储网络性能提升10倍

**目标**: 支持NVMe-oF Target和Initiator，实现高性能存储网络

| 子功能 | 描述 | 预计工时 | 负责 |
|--------|------|----------|------|
| NVMe-oF Target | 将本地NVMe设备导出为网络存储 | 12h | 兵部 |
| NVMe-oF Initiator | 连接远程NVMe-oF Target | 8h | 兵部 |
| RDMA支持 | 支持InfiniBand/RoCE高性能传输 | 10h | 工部 |
| 多路径支持 | 高可用连接，故障自动切换 | 6h | 工部 |
| WebUI配置 | Target/Initiator配置界面 | 4h | 兵部 |
| 性能监控 | NVMe-oF连接状态、性能指标 | 4h | 工部 |

**技术方案**:
- Linux内核 `nvmet` 和 `nvme-rdma` 模块
- 支持TCP和RDMA两种传输模式
- 兼容现有存储池架构

**里程碑**:
- M1 (04-05): Target/Initiator基础功能
- M2 (04-10): RDMA支持与多路径
- M3 (04-14): WebUI与性能监控

---

**背景**: 群晖DSM 7.3推出私有云AI服务，支持本地LLM

**目标**: 在本地部署AI能力，数据不出本地

| 子功能 | 描述 | 优先级 | 负责 |
|--------|------|--------|------|
| 本地LLM集成 | Ollama/vLLM本地推理服务 | P1 | 兵部 |
| AI去识别化 | 敏感信息智能识别与脱敏 | P1 | 刑部 |
| GPU加速支持 | NVIDIA/AMD/Intel GPU调度 | P1 | 工部 |
| OpenAI兼容API | 支持所有OpenAI兼容接口 | P1 | 兵部 |

**计划版本**: v2.290.0 (Q2 2026)

## 优先级调整说明

### 从v2.282.0调整

| 功能 | 原优先级 | 新优先级 | 原因 |
|------|----------|----------|------|
| 勒索软件检测 | P1 | **P0** | TrueNAS 26已发布，安全需求紧迫 |
| NVMe-oF支持 | P2 | **P0** | 企业用户高频需求，性能提升显著 |
| 私有云AI服务 | P0 | P1 | 技术复杂度高，延后至Q2 |
