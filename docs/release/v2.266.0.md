# NAS-OS v2.266.0 发布公告

**发布日期**: 2026-03-26  
**版本类型**: Stable

---

## 🎉 新功能介绍

### 内网穿透服务 (NatPierce)

v2.266.0 带来全新的内网穿透功能，让您随时随地安全访问 NAS：

- **P2P 直连模式**
  - 通过 STUN 协议实现 NAT 穿透
  - 点对点连接，低延迟高带宽
  - 支持 UDP 打洞技术

- **中继模式**
  - P2P 失败时自动回退
  - 通过中继服务器转发流量
  - 保证连接稳定性

- **智能模式选择**
  - 自动检测网络环境
  - 智能选择最优连接方式
  - 无缝切换，用户无感知

- **安全传输**
  - TLS 端到端加密
  - 认证令牌机制
  - IP 白名单访问控制

- **端口映射**
  - 支持 TCP/UDP 端口映射
  - 自动分配远程端口
  - 多端口同时映射

```bash
# 快速启用内网穿透
nasctl tunnel start

# 查看访问地址
nasctl tunnel info
# 输出: https://abc123.tunnel.nas-os.local

# 添加端口映射
nasctl tunnel port add --local 80 --remote 8080
```

### 存储管理增强

- **Hot Spare 优化**
  - 自动故障检测时间缩短 50%
  - 支持热备盘优先级设置
  - 重建进度实时显示

- **存储池健康监控**
  - 新增磁盘温度监控
  - 预测性故障告警
  - 历史健康趋势图表

## 📦 升级指南

### 从 v2.265.0 升级

#### 方式一：二进制文件升级

```bash
# 停止服务
sudo systemctl stop nas-os

# 备份配置
sudo cp -r /etc/nas-os /etc/nas-os.bak

# 下载新版本
wget https://github.com/crazyqin/nas-os/releases/download/v2.266.0/nasd-linux-amd64
chmod +x nasd-linux-amd64
sudo mv nasd-linux-amd64 /usr/local/bin/nasd

# 启动服务
sudo systemctl start nas-os

# 验证版本
nasd --version
```

#### 方式二：Docker 升级

```bash
# 拉取新镜像
docker pull ghcr.io/crazyqin/nas-os:v2.266.0

# 停止旧容器
docker stop nasd
docker rm nasd

# 启动新容器
docker run -d \
  --name nasd \
  --restart unless-stopped \
  -p 8080:8080 \
  -v /data:/data \
  -v /etc/nas-os:/config \
  ghcr.io/crazyqin/nas-os:v2.266.0

# 验证
docker logs nasd | head -5
```

### 新增配置项

v2.266.0 新增内网穿透配置文件 `/etc/nas-os/tunnel.yaml`：

```yaml
# 内网穿透配置
server:
  addr: "tunnel.nas-os.local"
  port: 443
  auth_token: ""  # 从管理后台获取

connection:
  mode: "auto"  # p2p, relay, auto
  heartbeat_interval: 30
  timeout: 30

port_mapping:
  enabled: true
  local_ports:
    - 80
    - 443
  allowed_ips: []

security:
  enable_tls: true
```

### 数据迁移

本次升级无需数据迁移，配置文件自动兼容旧版本。

## ⚠️ 已知问题

1. **部分运营商 NAT 不支持 P2P**
   - 症状：只能使用中继模式
   - 影响：连接延迟略高
   - 解决方案：系统自动回退到中继模式，不影响使用

2. **IPv6 环境下 P2P 可能失败**
   - 症状：IPv6-only 网络连接不稳定
   - 解决方案：临时禁用 IPv6 或使用中继模式
   - 计划修复：v2.267.0

3. **大量端口映射时启动变慢**
   - 症状：映射超过 20 个端口时启动耗时增加
   - 影响：服务启动延长 5-10 秒
   - 解决方案：减少不必要的端口映射

4. **ARMv7 内存占用**
   - 症状：内网穿透开启后内存增加约 30MB
   - 影响：低内存设备（<512MB）可能压力
   - 建议：内存紧张时关闭穿透或使用定时启用
