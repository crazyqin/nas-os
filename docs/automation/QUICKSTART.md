# 自动化工作流 - 快速入门

## 5 分钟上手指南

### 第一步：访问自动化页面

在浏览器中打开：`http://your-nas-ip/pages/automation.html`

### 第二步：创建第一个工作流

#### 方法 A：使用模板（推荐新手）

1. 滚动到页面底部的"预置模板"区域
2. 选择一个模板，例如"文件自动备份"
3. 点击模板卡片
4. 确认创建
5. 完成！工作流已启用

#### 方法 B：自定义工作流

1. 点击右上角"+ 新建工作流"
2. 从左侧面板拖拽一个触发器到画布：
   - 📁 文件变化 - 监控文件
   - ⏰ 定时任务 - 定时执行
   - 🔔 系统事件 - 响应事件
   - 🌐 Webhook - HTTP 回调

3. 继续拖拽动作到画布：
   - 📦 移动文件
   - 📋 复制文件
   - 🗑️ 删除文件
   - 🔄 格式转换
   - 📢 发送通知
   - 💻 执行命令

4. 点击右侧属性面板配置每个节点
5. 点击"保存"按钮

### 第三步：管理工作流

- **启用/禁用**: 点击工作流卡片上的"启用"或"禁用"按钮
- **编辑**: 点击"编辑"按钮修改工作流
- **删除**: 点击"删除"按钮移除工作流
- **手动执行**: 在 API 中调用执行端点测试工作流

## 常用场景示例

### 1. 自动备份下载文件

**目标**: 每天凌晨 2 点备份下载文件夹

**配置**:
- 触发器：定时任务 `0 2 * * *`
- 动作 1: 复制文件 `/home/user/downloads` → `/backup/downloads_{{timestamp}}`
- 动作 2: 发送通知 "备份完成"

### 2. 监控磁盘空间

**目标**: 每小时检查磁盘空间，不足时告警

**配置**:
- 触发器：定时任务 `0 * * * *`
- 动作 1: 执行命令 `df -h /`
- 动作 2: 如果空间<10%，发送告警通知

### 3. 图片自动压缩

**目标**: 上传的图片自动压缩

**配置**:
- 触发器：文件变化 `/uploads/*.jpg`
- 动作 1: 格式转换（压缩）
- 动作 2: 移动到压缩目录

### 4. 登录通知

**目标**: 用户登录时发送通知

**配置**:
- 触发器：系统事件 `user.login`
- 动作 1: 发送通知 "用户 {{username}} 已登录"

## 变量使用

在工作流配置中可以使用这些变量：

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{timestamp}}` | 当前时间戳 | `backup_20240311_120000` |
| `{{event.path}}` | 触发事件的文件路径 | `/home/user/file.txt` |
| `{{event.filename}}` | 触发事件的文件名 | `file.txt` |
| `{{event.username}}` | 触发事件的用户名 | `admin` |
| `{{workflow.id}}` | 工作流 ID | `wf_123456` |

## Cron 表达式速查

| 表达式 | 含义 |
|--------|------|
| `0 * * * *` | 每小时 |
| `0 0 * * *` | 每天午夜 |
| `0 2 * * *` | 每天凌晨 2 点 |
| `0 0 * * 0` | 每周日 |
| `0 0 1 * *` | 每月 1 号 |
| `*/5 * * * *` | 每 5 分钟 |
| `0 9-17 * * *` | 工作日 9 点到 17 点每小时 |

## 故障排除

### 工作流不执行？

1. ✅ 检查工作流是否已启用（绿色状态）
2. ✅ 验证触发器配置是否正确
3. ✅ 查看系统日志：`journalctl -u nasd`

### 动作执行失败？

1. ✅ 检查文件路径是否存在
2. ✅ 确认有足够的权限
3. ✅ 验证命令是否正确

### 需要帮助？

- 查看完整文档：`/docs/automation/README.md`
- 查看 API 文档：`GET /api/automation/templates`
- 检查日志：`/logs/automation.log`
