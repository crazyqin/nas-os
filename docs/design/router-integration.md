# 软路由集成设计

## 背景
飞牛fnOS支持QWRT软路由功能，NAS一键安装软路由实现一机多用。本设计研究nas-os软路由集成方案。

## 需求分析

### 功能目标
1. NAS系统内运行软路由服务
2. 支持主路由/旁路由模式
3. 网络隔离与安全防护
4. 与NAS存储服务共存

### 使用场景
- 家庭网络中心
- 小型办公网络
- IoT设备隔离
- VPN网关

## 架构设计

### 方案一：容器化软路由
```
┌─────────────────────────────────────┐
│           nas-os 主系统              │
│  ┌─────────────┐  ┌───────────────┐  │
│  │  NAS服务    │  │  软路由容器   │  │
│  │  SMB/NFS    │  │  OpenWRT      │  │
│  │  WebUI      │  │  iptables     │  │
│  └─────────────┘  └───────────────┘  │
│                                     │
│  ┌─────────────────────────────────┐│
│  │        Docker Network           ││
│  │    macvlan/bridge 模式          ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

### 方案二：系统级集成
- 直接在nas-os内核层实现路由功能
- iptables/nftables管理
- 与现有网络模块整合

## 技术选型

### 掯品对比
| 方案 | 飞牛fnOS | 群晖DSM | nas-os建议 |
|------|----------|---------|------------|
| 软路由引擎 | QWRT定制 | VPN Server | OpenWRT容器 |
| 网络模式 | 独立网口 | 虚拟网口 | macvlan |
| 防火墙 | iptables | iptables | nftables |

### 推荐方案：容器化OpenWRT
1. Docker容器运行OpenWRT镜像
2. macvlan网络直通物理网口
3. WebUI集成管理界面
4. 配置持久化存储

## API设计

```
/api/v1/router
  GET    /status         - 软路由状态
  POST   /enable         - 启用软路由
  POST   /disable        - 禁用软路由
  GET    /config         - 获取配置
  PUT    /config         - 更新配置
  POST   /restart        - 重启服务
```

## 实施计划

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| P1 | 容器化OpenWRT部署 | 高 |
| P1 | macvlan网络配置 | 高 |
| P2 | WebUI管理界面 | 中 |
| P2 | VPN集成 | 中 |
| P3 | 高级路由规则 | 低 |
