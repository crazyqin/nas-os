### 2.2 网络配置方案

#### 方案A：macvlan 模式（推荐）
- 软路由容器获得独立 MAC 地址
- 与 NAS 主机网络隔离
- 适合单网口场景

```yaml
# docker-compose.yml
services:
  openwrt:
    image: openwrt/rootfs:latest
    networks:
      macvlan_net:
        ipv4_address: 192.168.1.2
    privileged: true
    cap_add:
      - NET_ADMIN
      - NET_RAW

networks:
  macvlan_net:
    driver: macvlan
    driver_opts:
      parent: eth0
    ipam:
      config:
        - subnet: 192.168.1.0/24
          gateway: 192.168.1.1
```

#### 方案B：双网口模式
- eth0: WAN 口
- eth1: LAN 口
- NAS 作为网关

## 六、结论

软路由集成是 nas-os 差异化的重要功能，可以实现"一台设备，多种用途"：

1. **用户价值**: 节省硬件成本，一台 NAS 同时作为软路由
2. **技术可行**: Docker 容器化部署，风险可控
3. **竞争优势**: 对标飞牛fnOS，补齐功能短板

**建议**: 将软路由集成列为 v2.305.0 重点功能，优先实现基础部署能力。
