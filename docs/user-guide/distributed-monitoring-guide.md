# 分布式监控配置说明

## 概述

NAS-OS 分布式监控支持多节点集群的统一监控，提供集群健康评分、存储池监控、跨节点告警等功能。

## 功能特性

- **多节点管理**：添加、移除、监控集群节点
- **统一仪表板**：集中展示所有节点状态
- **存储池监控**：BTRFS/ZFS/MDADM 存储池健康
- **健康评分**：综合系统健康评估
- **告警聚合**：跨节点告警统一管理
- **Prometheus 集成**：标准监控指标导出

---

## 快速开始

### 1. 添加监控节点

#### 前提条件

- 所有节点已安装 NAS-OS
- 节点间网络互通
- 防火墙开放监控端口（默认 8080）

#### 添加步骤

1. 登录主节点 Web 管理界面
2. 进入 **监控** → **分布式监控** → **节点管理**
3. 点击 **添加节点**
4. 填写节点信息：
   - 节点名称（如 nas-node-1）
   - 节点地址（IP 或域名）
   - API 端口（默认 8080）
5. 点击 **测试连接**
6. 连接成功后点击 **保存**

### 2. 验证节点状态

添加成功后，节点列表显示：

- **状态**：活跃/离线
- **角色**：主节点/从节点
- **最后心跳**：最近通信时间
- **健康评分**：节点综合健康度

## 监控仪表板

### 系统概览

仪表板顶部显示集群整体状态：

- 活跃节点数
- 总存储容量
- 总使用率
- 告警数量
- 集群健康评分

### 节点详情

点击节点查看详细信息：

#### 系统指标

| 指标 | 说明 | 健康阈值 |
|------|------|----------|
| CPU 使用率 | CPU 占用百分比 | < 80% |
| 内存使用率 | 内存占用百分比 | < 85% |
| 负载均衡 | 1/5/15 分钟负载 | < 核心数 |
| 运行时间 | 系统启动时长 | - |

#### 存储指标

| 指标 | 说明 | 健康阈值 |
|------|------|----------|
| 存储池使用率 | 各存储池占用 | < 85% |
| IO 延迟 | 读写延迟 | < 10ms |
| 重建进度 | RAID 重建状态 | - |
| 健康设备 | 正常磁盘数量 | = 设备总数 |

#### 网络指标

| 指标 | 说明 |
|------|------|
| 入站带宽 | 接收速率 |
| 出站带宽 | 发送速率 |
| 连接数 | 活跃连接 |

## 存储池监控

### 支持的存储类型

| 类型 | 说明 | 监控项目 |
|------|------|----------|
| BTRFS | B-tree 文件系统 | 使用率、RAID 状态、Scrub 状态 |
| ZFS | Zettabyte 文件系统 | 使用率、VDEV 状态、Scrub |
| MDADM | Linux 软件 RAID | 使用率、阵列状态、同步进度 |

### 存储池状态

| 状态 | 说明 | 建议 |
|------|------|------|
| healthy | 健康 | 无需操作 |
| degraded | 降级 | 检查故障磁盘 |
| rebuilding | 重建中 | 等待完成 |
| failed | 失败 | 立即处理 |

### 存储池告警

自动告警触发条件：

- 使用率超过 85%（警告）
- 使用率超过 95%（严重）
- 设备故障
- 重建进度异常

## 告警管理

### 告警规则配置

1. 进入 **监控** → **告警规则**
2. 预置规则：

| 规则名 | 类型 | 阈值 | 级别 |
|--------|------|------|------|
| cpu-warning | CPU | 80% | 警告 |
| cpu-critical | CPU | 95% | 严重 |
| memory-warning | 内存 | 85% | 警告 |
| memory-critical | 内存 | 95% | 严重 |
| disk-warning | 磁盘 | 85% | 警告 |
| disk-critical | 磁盘 | 95% | 严重 |

### 自定义告警规则

1. 点击 **添加规则**
2. 配置：
   - 规则名称
   - 监控类型
   - 阈值
   - 告警级别
   - 通知方式

### 告警通知

支持的通知方式：

- 系统通知（Web 界面）
- 邮件通知
- Webhook
- WebSocket 实时推送

## Prometheus 集成

### 启用 Prometheus 指标

1. 进入 **监控** → **设置** → **Prometheus**
2. 开启 **启用 Prometheus 端点**
3. 配置访问地址：`/api/v1/monitor/prometheus/metrics`

### 可用指标

```
# CPU 指标
nas_cpu_usage{host="nas-node-1"} 35.2
nas_cpu_cores{host="nas-node-1"} 8

# 内存指标
nas_memory_usage{host="nas-node-1"} 62.5
nas_memory_total_bytes{host="nas-node-1"} 8589934592

# 磁盘指标
nas_disk_usage_percent{host="nas-node-1",device="sda"} 50.0
nas_disk_io_latency_ms{host="nas-node-1",device="sda"} 2.5

# 存储池指标
nas_pool_usage_percent{host="nas-node-1",pool="main"} 45.0
nas_pool_health_score{host="nas-node-1",pool="main"} 95
```

### Grafana 仪表板

可导入 NAS-OS 提供的 Grafana 仪表板模板：

1. 下载仪表板 JSON
2. 在 Grafana 中导入
3. 配置 Prometheus 数据源

## 健康评分

### 评分算法

系统综合以下因素计算健康评分：

| 组件 | 权重 | 评分依据 |
|------|------|----------|
| CPU | 20% | 使用率、负载 |
| 内存 | 20% | 使用率、交换分区 |
| 存储 | 40% | 使用率、健康状态、IO |
| 网络 | 20% | 连接性、带宽 |

### 评分等级

| 分数范围 | 状态 | 说明 |
|----------|------|------|
| 90-100 | 健康 | 系统运行良好 |
| 70-89 | 注意 | 存在潜在问题 |
| 50-69 | 警告 | 需要关注 |
| 0-49 | 严重 | 需要立即处理 |

### 改进建议

系统根据健康评分提供改进建议：

- 资源优化建议
- 安全加固建议
- 性能调优建议

- 识别性能瓶颈
- 平衡负载分布

## 故障排除

### Q: 节点显示离线？

1. 检查节点网络连通性
2. 确认 API 服务运行正常
3. 检查防火墙规则
4. 验证认证信息

### Q: 指标数据不准确？

1. 检查节点时间同步
2. 重启监控服务
3. 清除缓存数据

### Q: 告警未触发？

1. 检查告警规则是否启用
2. 确认阈值配置正确
3. 验证通知渠道配置

## API 调用示例

### 获取集群节点状态

```bash
curl -X GET "https://nas.local/api/v1/monitor/cluster/nodes" \
  -H "Authorization: Bearer <token>"
```

### 获取节点指标

```bash
curl -X GET "https://nas.local/api/v1/monitor/cluster/nodes/node-001/metrics" \
  -H "Authorization: Bearer <token>"
```

### 获取存储池指标

```bash
curl -X GET "https://nas.local/api/v1/monitor/storage/pools" \
  -H "Authorization: Bearer <token>"
```

### 获取健康评分

```bash
curl -X GET "https://nas.local/api/v1/monitor/health/score" \
  -H "Authorization: Bearer <token>"
```

更多 API 详情请参考 [监控 API 文档](../api/monitor-api.md)。
