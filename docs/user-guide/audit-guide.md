# 审计模块使用指南

**版本**: v2.61.0  
**更新日期**: 2026-03-15

## 概述

NAS-OS 审计模块提供全面的安全审计功能，帮助管理员追踪系统操作、满足合规要求、检测安全威胁。

## 功能特性

- **多分类审计日志**：认证、访问、数据、系统、安全等
- **合规报告生成**：支持 GDPR、等级保护、ISO 27001 等标准
- **日志完整性保护**：数字签名防篡改
- **灵活的导出选项**：JSON、CSV、XML 格式
- **实时统计仪表板**

---

## 快速开始

### 1. 访问审计日志

1. 登录 NAS-OS Web 管理界面
2. 进入 **系统** → **审计日志**
3. 默认显示最近 24 小时的日志

### 2. 筛选日志

使用顶部工具栏进行筛选：

- **时间范围**：选择预设范围或自定义时间
- **日志级别**：信息、警告、错误、严重
- **日志分类**：认证、访问、数据、系统、安全
- **用户/IP**：按用户名或 IP 地址过滤
- **关键词**：全文搜索

### 3. 查看日志详情

点击任意日志条目查看完整详情，包括：

- 操作时间
- 操作用户
- 来源 IP
- 操作状态
- 详细信息
- 数字签名（如启用）

---

## 日志分类说明

| 分类 | 说明 | 示例事件 |
|------|------|----------|
| auth | 认证相关 | 登录、登出、密码修改 |
| access | 访问控制 | 文件访问、共享连接 |
| data | 数据操作 | 文件读写、删除、移动 |
| system | 系统配置 | 服务启停、配置变更 |
| security | 安全事件 | 入侵检测、权限提升 |
| compliance | 合规相关 | 合规检查、报告生成 |

## 合规报告

### 生成合规报告

1. 进入 **审计日志** → **合规报告**
2. 选择合规标准：
   - **GDPR** - 欧盟通用数据保护条例
   - **等级保护** - 中国网络安全等级保护
   - **ISO 27001** - 信息安全管理体系
   - **HIPAA** - 美国健康保险携带和责任法案
   - **PCI DSS** - 支付卡行业数据安全标准
3. 选择报告周期
4. 点击 **生成报告**

### 报告内容

合规报告包含：

- **摘要统计**：事件总数、失败认证、数据访问等
- **合规发现**：潜在风险和异常行为
- **改进建议**：安全加固建议

---

## 日志导出

### 导出步骤

1. 设置筛选条件
2. 点击 **导出** 按钮
3. 选择导出格式：
   - **JSON** - 适合程序处理
   - **CSV** - 适合 Excel 分析
   - **XML** - 适合系统集成
4. 选择是否包含数字签名
5. 点击 **下载**

---

## 完整性验证

### 验证日志完整性

1. 进入 **审计日志** → **完整性验证**
2. 点击 **开始验证**
3. 系统将检查所有日志的数字签名

### 验证结果

- **通过**：所有日志完整无篡改
- **警告**：发现签名不一致的日志（可能被篡改）

## 配置审计模块

### 访问配置

1. 进入 **系统设置** → **审计配置**
2. 可配置项：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用审计 | 开启/关闭审计功能 | 开启 |
| 日志存储路径 | 审计日志存储位置 | /var/lib/nas-os/audit |
| 最大日志条数 | 保存的最大日志数量 | 1,000,000 |
| 最大保留天数 | 日志保留时间 | 365 天 |
| 启用数字签名 | 防止日志篡改 | 开启 |
| 启用压缩 | 压缩历史日志 | 开启 |

### 保留策略

可为不同分类设置不同的保留策略：

1. 点击 **添加保留策略**
2. 选择日志分类
3. 设置保留天数和最大条数
4. 选择是否压缩

## 最佳实践

### 1. 定期审查日志

- 每周审查安全相关日志
- 关注失败认证和异常访问
- 设置告警阈值

### 2. 合规准备

- 了解适用的合规标准
- 定期生成合规报告
- 保存报告记录

### 3. 安全加固

- 启用数字签名保护日志
- 设置合理的保留期限
- 定期导出备份日志

### 4. 性能优化

- 对于高流量系统，调整保存间隔
- 启用压缩减少存储占用
- 合理设置保留策略

## 故障排除

### Q: 日志不显示？

1. 检查审计功能是否启用
2. 检查日志存储路径是否有写权限
3. 查看系统日志 `/var/log/nas-os/audit.log`

### Q: 导出失败？

1. 检查磁盘空间
2. 减少导出时间范围
3. 检查浏览器下载权限

### Q: 完整性验证失败？

1. 检查是否启用了数字签名
2. 检查系统时间是否正确
3. 查看详细错误信息定位问题日志

## API 调用示例

### 查询审计日志

```bash
curl -X GET "https://nas.local/api/v1/audit/logs?limit=20&category=auth" \
  -H "Authorization: Bearer <token>"
```

### 导出日志

```bash
curl -X GET "https://nas.local/api/v1/audit/export?format=csv" \
  -H "Authorization: Bearer <token>" \
  -o audit-logs.csv
```

### 生成合规报告

```bash
curl -X GET "https://nas.local/api/v1/audit/compliance/report?standard=gdpr" \
  -H "Authorization: Bearer <token>"
```

更多 API 详情请参考 [审计 API 文档](../api/audit-api.md)。
