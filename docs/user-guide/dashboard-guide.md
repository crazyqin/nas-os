# 监控仪表板使用指南

> **版本**: v2.56.0  
> **更新日期**: 2026-03-15

---

## 概述

NAS-OS 监控仪表板提供实时系统监控和可视化功能，帮助您全面了解系统运行状态。

### 主要功能

- **实时指标监控** - CPU、内存、磁盘、网络等核心指标实时更新
- **自定义布局** - 支持拖拽式小组件排列，个性化仪表板
- **多种小组件** - 提供 10+ 种监控小组件类型
- **历史趋势** - 支持 1h/6h/24h/7d 多时间范围查看
- **健康评分** - 系统健康度综合评估与可视化
- **告警集成** - 与告警系统联动，异常状态高亮显示

---

## 快速开始

### 访问仪表板

1. 登录 NAS-OS Web 界面
2. 点击左侧导航栏「仪表板」
3. 默认显示系统概览视图

### 默认仪表板

系统提供以下默认仪表板：

| 仪表板 | 说明 |
|--------|------|
| 系统概览 | CPU、内存、磁盘、网络综合视图 |
| 存储监控 | 卷使用率、I/O 性能、健康状态 |
| 网络监控 | 带宽使用、连接状态、端口流量 |
| 容器监控 | 容器资源占用、运行状态 |

---

## 小组件类型

### 系统资源类

| 小组件 | 说明 | 默认刷新 |
|--------|------|----------|
| CPU 使用率 | 实时 CPU 使用百分比 | 5s |
| 内存使用 | 内存使用量和百分比 | 5s |
| 磁盘 I/O | 读写速度和 IOPS | 10s |
| 网络流量 | 上传/下载速度 | 5s |

### 存储类

| 小组件 | 说明 | 默认刷新 |
|--------|------|----------|
| 卷使用率 | 各存储卷空间使用情况 | 30s |
| 磁盘温度 | 硬盘温度监控 | 60s |
| SMART 状态 | 磁盘健康状态 | 300s |

### 服务类

| 小组件 | 说明 | 默认刷新 |
|--------|------|----------|
| 服务状态 | 系统服务运行状态 | 30s |
| 容器状态 | Docker 容器运行状态 | 10s |

## 自定义仪表板

### 添加小组件

1. 点击右上角「编辑」按钮进入编辑模式
2. 点击「添加小组件」
3. 选择小组件类型
4. 配置小组件参数：
   - 标题
   - 数据源
   - 刷新间隔
   - 显示范围
5. 点击「保存」

### 调整布局

1. 进入编辑模式
2. 拖拽小组件到目标位置
3. 拖拽小组件边缘调整大小
4. 点击「保存布局」

### 删除小组件

1. 进入编辑模式
2. 点击小组件右上角「×」删除按钮
3. 确认删除
4. 点击「保存」

## 告警配置

### 设置告警阈值

1. 进入小组件设置
2. 切换到「告警」标签
3. 配置告警规则：
   - 警告阈值（黄色）
   - 严重阈值（红色）
   - 持续时间
4. 选择通知方式：
   - 邮件
   - 钉钉/企业微信
   - Webhook
5. 保存配置

### 告警示例

```yaml
# CPU 使用率告警
cpu_alert:
  warning: 70    # 超过 70% 警告
  critical: 90   # 超过 90% 严重
  duration: 5m   # 持续 5 分钟触发
  notify:
    - email
    - dingtalk
```

## 历史数据查看

### 时间范围

仪表板支持以下时间范围：

- **1 小时** - 高精度数据，1 分钟间隔
- **6 小时** - 中精度数据，5 分钟间隔
- **24 小时** - 日视图，15 分钟间隔
- **7 天** - 周视图，1 小时间隔

### 数据导出

1. 点击小组件右上角「导出」按钮
2. 选择导出格式：
   - CSV
   - JSON
   - Excel
3. 选择时间范围
4. 点击「下载」

## 健康评分

### 评分维度

系统健康评分基于以下维度计算：

| 维度 | 权重 | 说明 |
|------|------|------|
| CPU 负载 | 20% | CPU 使用率和负载均衡 |
| 内存状态 | 20% | 内存使用率和交换空间 |
| 存储健康 | 25% | 磁盘空间和 SMART 状态 |
| 网络状态 | 15% | 网络连通性和带宽 |
| 服务状态 | 20% | 关键服务运行状态 |

### 评分等级

| 分数 | 等级 | 状态 |
|------|------|------|
| 90-100 | 优秀 | 🟢 系统运行良好 |
| 70-89 | 良好 | 🟡 部分指标需关注 |
| 50-69 | 一般 | 🟠 存在潜在问题 |
| 0-49 | 异常 | 🔴 需要立即处理 |

## 最佳实践

### 仪表板设计建议

1. **分层设计** - 创建概览 → 详情的层次结构
2. **关键指标优先** - 将最重要的指标放在显眼位置
3. **合理刷新** - 根据数据变化频率设置刷新间隔
4. **告警适度** - 避免过多告警导致疲劳

### 性能优化

1. 减少实时小组件数量
2. 适当延长刷新间隔
3. 合理设置历史数据保留周期

### 安全建议

1. 不要在公共网络暴露仪表板
2. 定期检查访问日志
3. 为敏感操作启用二次验证

## 常见问题

### Q: 仪表板数据不更新？

检查：
1. WebSocket 连接是否正常
2. 后端服务是否运行
3. 浏览器控制台是否有错误

### Q: 如何添加自定义监控指标？

目前支持通过 API 添加自定义数据源，详见 [Dashboard API 文档](../api/dashboard-api.md)。

### Q: 历史数据保留多久？

默认配置：
- 原始数据：7 天
- 聚合数据：30 天
- 可通过配置文件调整
