## 一、功能概述

### 1.1 什么是RAIDZ扩展

RAIDZ扩展（RAIDZ Expansion）是一项存储池在线扩容技术，允许用户向现有RAIDZ阵列逐个添加磁盘，实现增量扩容，无需重建整个存储池。

### 1.2 核心价值

| 特性 | 价值说明 |
|------|----------|
| **在线操作** | 扩容过程中存储池保持可访问，业务不中断 |
| **增量扩容** | 单盘逐个添加，降低扩容门槛 |
| **保持冗余** | 原有RAIDZ1/RAIDZ2/RAIDZ3冗余级别不变 |
| **数据重分布** | 自动将数据重新条带化至所有磁盘 |
| **中断恢复** | 支持系统重启后继续扩展过程 |

### 1.3 竞品对标

**对标产品**: TrueNAS 24.10 Electric Eel

| 对比项 | TrueNAS RAIDZ Expansion | nas-os规划 |
|--------|-------------------------|-----------|
| 技术基础 | OpenZFS 2.3 | btrfs多设备管理 |
| 扩容速度 | 5-10倍加速（优化后） | 待评估优化 |
| 冗余级别 | RAIDZ1/2/3 | RAID1/10优先 |
| 中断恢复 | 自动 | 需封装实现 |
| WebUI支持 | 完善 | 规划开发 |

## 二、技术背景

### 2.1 竞品技术分析

#### TrueNAS RAIDZ Expansion（OpenZFS 2.3）

**工作原理**：
1. 使用`zpool attach`命令添加新磁盘
2. 系统自动进行数据重新分布（reflow）
3. 数据在所有磁盘间重新条带化
4. 新数据使用新的条带宽度写入

**TrueNAS优化成果**：
- HDD阵列：5倍加速（典型情况）
- 最大可达：10倍加速
- 原始数据：6.38GB在4分31秒内完成重分布

**开发投入**：
- 开发周期：约3年
- 资金投入：约$100,000
- 核心开发者：Matt Ahrens（ZFS联合创始人）

### 2.2 nas-os技术路径

#### btrfs扩展能力现状

**优势**：
- 支持动态添加/移除/替换设备
- RAID1/RAID10扩展已稳定
- 支持Profile动态转换

**限制**：
- RAID5/6状态：不稳定（官方标记）
- 扩展需多步骤：添加设备 + balance操作
- Balance过程对性能影响显著

**技术调研报告**: 见 `docs/RAIDZ_EXPANSION_RESEARCH.md`

### 2.3 实现方案评估

| 方案 | 适用场景 | 开发成本 | 稳定性 |
|------|----------|----------|--------|
| 方案A：btrfs优化封装 | RAID1/RAID10用户 | 低 | 高 |
| 方案B：引入ZFS后端 | 企业用户 | 高 | 最高 |
| 方案C：混合双引擎 | 全场景覆盖 | 中 | 中 |

**推荐策略**：
- **短期（v3.x）**：保持btrfs，优化RAID1/RAID10扩展体验
- **中期（v4.x）**：引入ZFS作为企业版可选后端
- **长期（v5.x）**：根据用户反馈决定主要引擎

## 三、用户使用场景

### 3.1 家庭用户场景

**场景描述**：
用户现有4盘位NAS，使用3块4TB硬盘组成RAID1阵列，存储容量6TB。随着数据增长，需要扩容至12TB。

**扩容流程（规划）**：
1. 购买1块新4TB硬盘
2. 通过WebUI"存储池扩容"功能添加
3. 系统自动执行数据重分布
4. 扩容完成，容量翻倍

**预计耗时**：
- 6TB数据重分布：约2-4小时（HDD）
- 业务中断：0分钟（在线操作）

### 3.2 小型企业场景

**场景描述**：
小型工作室使用8盘位NAS，现有6块8TB硬盘组成RAID10阵列，存储容量24TB。业务增长需要扩容至32TB。

**扩容方案**：
- 方案1：添加2块8TB硬盘，保持RAID10
- 方案2：转换为RAID1C3，提高冗余级别

### 3.3 技术爱好者场景

**场景描述**：
技术爱好者希望体验ZFS级别的数据完整性保证，同时保持btrfs的灵活性。

**解决方案**：
- 中期版本提供ZFS可选后端
- 提供存储池迁移工具
- 保持双引擎支持

## 五、风险与限制

### 5.1 技术风险

| 风险项 | 等级 | 缓解措施 |
|--------|------|----------|
| btrfs RAID5/6不稳定 | 高 | 禁用或强烈警告，推荐RAID1/10 |
| Balance过程长 | 中 | 后台调度+限速+进度显示 |
| 扩容中断数据风险 | 中 | 完整备份+验证机制 |
| ZFS许可证兼容 | 中 | 法律咨询+可选安装 |

### 5.2 用户限制

**当前限制（基于btrfs）**：
- RAID5/6不建议用于生产环境
- 扩容过程需手动触发balance
- 重分布过程对性能有影响

**规划改进**：
- 封装自动化流程
- 智能调度降低影响
- 明确用户提示与风险说明
