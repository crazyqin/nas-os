## 3. 成本优化策略

### 3.1 本地 vs 云端决策矩阵

| 日均调用量 | 推荐方案 | 月成本估算 |
|------------|----------|------------|
| <100次 | 云端API (turbo模型) | <50元 |
| 100-1000次 | 云端API (混合turbo+标准) | 50-300元 |
| 1000-10000次 | 本地GPU (RTX 4090) | 300-500元 |
| >10000次 | 本地GPU集群 | 按需扩展 |

### 3.2 NAS-OS推荐配置

基于本项目AI服务模块分析：

```yaml
# 生产环境推荐配置
ai_service:
  primary_provider: local      # 主推理本地化
  fallback_provider: baidu     # 云端作为备用
  
  local_config:
    model: qwen2-7b-instruct   # 本地模型
    gpu: rtx_4090              # 硬件
    max_batch_size: 8
    
  cloud_config:
    provider: baidu
    model: ernie-3.5           # 经济模型
    daily_limit: 100           # 云端日限额
    
  cost_target:
    monthly_budget: 500        # 月预算(元)
    per_request_target: 0.05   # 单次成本目标
```

### 3.3 成本节约建议

1. **模型分层**: 日常任务用本地7B模型，复杂任务用云端大模型
2. **请求合并**: 批量处理减少API调用次数
3. **缓存策略**: 相似请求复用结果，减少重复计算
4. **精简提示词**: 减少输入token消耗

## 5. 存储成本与勒索检测关联

### 勒索检测存储需求

| 功能 | 存储需求 | 成本影响 |
|------|----------|----------|
| 特征库存储 | ~100MB | 固定低成本 |
| 隔离区存储 | 配置10GB上限 | 需监控 |
| 历史事件日志 | 按事件数增长 | 需定期清理 |
| 自动快照备份 | 按触发频率增长 | 潜在高成本 |

### 存储成本估算公式

```go
// 勒索检测相关存储成本
TotalRansomwareCost = 
    SignatureDBCost (固定 ~1元/月) +
    QuarantineCost (按使用量计费) +
    EventLogCost (需定期清理策略) +
    SnapshotBackupCost (按触发次数)
```

### 优化建议
1. **隔离区自动清理**: 设置30天过期自动删除
2. **事件日志压缩**: 保留30天详细日志，之后压缩归档
3. **快照智能触发**: 仅在检测到威胁时创建快照
