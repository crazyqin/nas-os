## 1. 概述

本报告针对nas-os项目计划新增的隐私相册功能进行数据合规审查，分析数据加密要求、用户隐私保护方案及相关法规遵循建议。

## 2. 适用法规框架

### 2.1 主要法规

| 法规 | 适用范围 | 生效日期 |
|------|----------|----------|
| 《中华人民共和国个人信息保护法》 | 中国境内 | 2021年11月1日 |
| 《中华人民共和国数据安全法》 | 中国境内 | 2021年9月1日 |
| 《中华人民共和国网络安全法》 | 中国境内 | 2017年6月1日 |
| 《信息安全技术 个人信息安全规范》(GB/T 35273-2020) | 国家标准 | 2020年10月1日 |

### 2.2 核心原则

根据《个人信息保护法》第六条，处理个人信息应遵循以下原则：

1. **合法、正当、必要原则**
2. **最小必要原则** - 收集范围限于处理目的最小范围
3. **公开透明原则**
4. **质量保证原则** - 确保信息准确、完整、及时
5. **安全保障原则**

## 3. 数据加密合规要求

### 3.1 加密技术标准

根据《数据安全法》和《网络安全法》，隐私相册数据应满足以下加密要求：

| 数据类型 | 加密要求 | 推荐方案 |
|----------|----------|----------|
| 静态存储数据 | 强制加密 | AES-256-GCM |
| 传输中数据 | 强制加密 | TLS 1.3 |
| 密钥材料 | 独立加密存储 | HSM/密钥管理系统 |
| 元数据 | 敏感字段加密 | AES-256 |

### 3.2 加密技术选型建议

#### 3.2.1 存储加密

```
推荐方案：
- 算法：AES-256-GCM（认证加密）
- 密钥派生：Argon2id 或 PBKDF2（迭代次数≥100,000）
- 密钥存储：用户密码派生 + 可选硬件密钥
```

#### 3.2.2 密钥管理

| 方案 | 安全级别 | 用户体验 | 推荐场景 |
|------|----------|----------|----------|
| 用户密码派生 | 高 | 需记住密码 | 默认方案 |
| 生物识别+密钥库 | 高 | 便捷 | 移动端优先 |
| 硬件安全模块 | 最高 | 需硬件支持 | 企业级 |
| 云端密钥托管 | 中 | 最便捷 | 需谨慎评估 |

### 3.3 加密实施要点

| 要点 | 要求 | 合规依据 |
|------|------|----------|
| 密钥不可恢复 | 系统不应存储可还原的主密钥 | 最小权限原则 |
| 密码不明文存储 | 仅存储派生验证值 | 《网络安全法》第21条 |
| 加密算法选择 | 使用国家认可的加密算法 | 《密码法》 |
| 国密支持（可选） | 考虑SM4等国密算法 | 政企场景 |

## 4. 用户隐私保护方案

### 4.1 数据分类分级

隐私相册涉及的个人信息分类：

| 分类 | 数据项 | 敏感程度 | 保护要求 |
|------|--------|----------|----------|
| 生物识别 | 人脸、指纹 | 极高 | 单独同意，明示告知 |
| 身份证件 | 身份证、护照 | 极高 | 单独同意，最小存储 |
| 个人影像 | 照片、视频 | 高 | 加密存储，访问控制 |
| 行踪轨迹 | 位置信息、时间戳 | 高 | 加密或匿名化 |
| 设备信息 | 设备ID、IP | 中 | 匿名化处理 |

### 4.2 隐私保护技术措施

#### 4.2.1 访问控制

```
必须实现：
├── 身份认证
│   ├── 多因素认证（推荐）
│   ├── 生物识别（可选）
│   └── 密码强度策略
├── 访问日志
│   ├── 访问时间记录
│   ├── 访问设备记录
│   └── 操作行为记录
└── 会话管理
    ├── 自动锁定机制
    ├── 超时登出
    └── 异常登录提醒
```

#### 4.2.2 数据最小化

| 原则 | 实施措施 |
|------|----------|
| 最小收集 | 仅收集功能必需的元数据 |
| 最小存储 | 设置自动删除/匿名化策略 |
| 最小访问 | 基于角色的访问控制 |
| 最小暴露 | 界面脱敏显示 |

#### 4.2.3 隐私设计（Privacy by Design）

| 设计原则 | 隐私相册应用 |
|----------|--------------|
| 主动预防 | 默认启用加密，而非可选 |
| 默认隐私 | 新相册默认私密，需用户主动公开 |
| 嵌入设计 | 隐私保护作为核心功能而非附加 |
| 全生命周期 | 从创建到删除全程保护 |
| 端到端安全 | 本地加密，服务器不可见明文 |
| 可见透明 | 隐私设置清晰可见 |
| 用户中心 | 用户可完全控制自己的数据 |

### 4.3 用户权利保障

根据《个人信息保护法》第四章，用户享有以下权利：

| 用户权利 | 系统实现要求 |
|----------|--------------|
| 知情权 | 明确告知数据收集、存储、使用方式 |
| 决定权 | 用户可选择是否使用隐私相册功能 |
| 查询权 | 用户可查看自己存储的所有数据 |
| 更正权 | 用户可修改、补充个人信息 |
| 删除权 | 用户可删除相册，系统需彻底清除 |
| 导出权 | 用户可导出自己的照片数据 |
| 撤回同意权 | 用户可随时撤回授权，停止使用 |

## 5. 数据生命周期管理

### 5.1 数据收集阶段

```
合规清单：
□ 仅在用户创建隐私相册时收集数据
□ 明确告知收集目的、方式、范围
□ 获取用户明示同意（非默认勾选）
□ 不收集与功能无关的数据
□ 记录同意时间和版本
```

### 5.2 数据存储阶段

| 要求 | 实施 |
|------|------|
| 存储位置 | 明确告知服务器位置 |
| 存储期限 | 明确告知保留时间或策略 |
| 安全存储 | 加密存储，访问控制 |
| 备份管理 | 加密备份，定期测试恢复 |
| 日志留存 | 操作日志保留不少于6个月 |

### 5.3 数据使用阶段

| 场景 | 合规要求 |
|------|----------|
| 相册展示 | 仅展示用户自己的数据 |
| 搜索功能 | 本地搜索，不上传索引 |
| 分享功能 | 需用户明确授权，有时间限制 |
| AI识别 | 明确告知，提供禁用选项 |
| 缩略图生成 | 本地生成，不发送原图 |

### 5.4 数据删除阶段

```
删除流程（硬删除）：
1. 用户发起删除请求
2. 验证用户身份
3. 删除主存储数据
4. 删除所有备份副本
5. 删除索引和元数据
6. 删除操作日志中的敏感内容
7. 记录删除操作（不含敏感数据）
```

## 9. 相关法规摘录

### 9.1 《个人信息保护法》关键条款

**第六条** 处理个人信息应当具有明确、合理的目的，并应当与处理目的直接相关。收集个人信息，应当限于实现处理目的的最小范围，不得过度收集个人信息。

**第十三条** 符合下列情形之一的，个人信息处理者方可处理个人信息：
（一）取得个人的同意；

**第二十八条** 敏感个人信息是一旦泄露或者非法使用，容易导致自然人人格尊严受到侵害或者人身、财产安全受到危害的个人信息，包括生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等信息，以及不满十四周岁未成年人的个人信息。
只有在具有特定的目的和充分的必要性，并采取严格保护措施的情形下，个人信息处理者方可处理敏感个人信息。

### 9.2 《数据安全法》关键条款

**第二十七条** 开展数据处理活动应当依照法律、法规的规定，建立健全全流程数据安全管理制度，组织开展数据安全教育培训，采取相应的技术措施和其他必要措施，保障数据安全。

### B.2 密钥派生参数建议

```
PBKDF2：
- 迭代次数：≥ 100,000
- 盐值长度：≥ 128 bits
- 输出长度：256 bits
- 哈希算法：SHA-256 或 SHA-512

Argon2id（推荐）：
- 迭代次数：≥ 3
- 内存：≥ 64 MB
- 并行度：≥ 4
- 输出长度：256 bits
```
