## 3. 发现的安全问题

### 3.1 中等风险

#### [M-01] 路径遍历风险 - handlers.go

**位置**: `internal/storage/handlers.go`  
**严重程度**: 中等

**描述**: 多个 API 端点接受用户输入的路径参数，未进行路径遍历验证。

```go
// 例如 deleteVolume 直接使用 URL 参数
name := c.Param("name")
```

**影响**: 攻击者可能通过构造特殊路径访问非预期资源。

**建议修复**:
```go
// 添加路径验证函数
func validatePathParam(name string) error {
    if strings.Contains(name, "..") || strings.ContainsAny(name, "/\\") {
        return fmt.Errorf("invalid path parameter")
    }
    return nil
}
```

---

#### [M-02] 命令注入风险 - smart_monitor.go

**位置**: `internal/storage/smart_monitor.go:400`  
**严重程度**: 中等

**描述**: 使用 `exec.Command` 执行外部命令，虽然有基本验证，但验证逻辑分散。

**现有保护**:
```go
// 已有验证（第395-399行）
if device == "" || strings.ContainsAny(device, ";|&$`()<>") {
    return nil, fmt.Errorf("无效的设备路径")
}
if !strings.HasPrefix(device, "/dev/") {
    return nil, fmt.Errorf("设备路径必须以 /dev/ 开头")
}
```

**建议**: 将验证逻辑抽取为独立函数，统一应用到所有命令执行点。

---

#### [M-03] 缺乏速率限制 - handlers.go

**位置**: `internal/storage/handlers.go`  
**严重程度**: 中等

**描述**: 所有 API 端点缺乏速率限制，可能遭受 DoS 攻击。

**建议**: 添加基于 IP 和用户的速率限制中间件。

---

### 3.2 低风险

#### [L-01] 敏感信息可能泄露 - manager.go

**位置**: `internal/storage/manager.go:353`  
**严重程度**: 低

**描述**: `DeleteVolume` 使用 `os.RemoveAll` 删除挂载点，删除失败时忽略错误。

```go
_ = os.RemoveAll(vol.MountPoint)
```

**建议**: 记录删除失败的错误，便于审计追踪。

---

#### [L-02] 缺乏输入长度限制 - handlers.go

**位置**: `internal/storage/handlers.go`  
**严重程度**: 低

**描述**: 请求体字段缺乏长度限制，可能导致资源耗尽。

**建议**: 添加最大长度验证：
```go
type CreateVolumeRequest struct {
    Name    string   `json:"name" binding:"required,max=64"`
    Devices []string `json:"devices" binding:"required,min=1,max=64"`
    Profile string   `json:"profile" binding:"omitempty,oneof=single raid0 raid1 raid5 raid6 raid10"`
}
```
