### 2.2 路径遍历风险 (G703)

**严重程度: 高** | **置信度: 高**

主要位置:
- `internal/webdav/server.go` - 35+ 处报告
- `plugins/filemanager-enhance/main.go` - 1 处
- `internal/vm/snapshot.go` - 1 处

**实际风险评估**: **低风险（大部分为误报）**

代码已有防护措施:
```go
// resolvePath 函数实现
if strings.Contains(decodedPath, "..") {
    return "", ErrPathTraversal
}
// 路径前缀验证
if !strings.HasPrefix(absFullPath, absBasePath) {
    return "", ErrPathTraversal
}
```

### 2.3 TLS 证书验证绕过 (G402)

**严重程度: 高** | **置信度: 低**

位置:
- `internal/auth/ldap.go:153`
- `internal/ldap/client.go:73, 107`

**实际风险评估**: **可控**

代码有环境限制:
```go
skipVerify := config.SkipTLSVerify && os.Getenv("ENV") == "test"
```

仅在测试环境 (`ENV=test`) 允许跳过验证，生产环境默认验证证书。

## 三、中等风险问题

### 3.1 文件权限过宽 (G306)

**数量**: 104 处

大部分配置文件使用 `0644` 权限，对于非敏感数据可接受，但以下位置建议改为 `0600`:
- 包含敏感配置的文件
- 用户会话数据
- 认证相关配置

### 3.2 弱加密算法 (G505)

位置:
- `internal/network/ddns_providers.go:5` - 使用 SHA1 进行签名
- `internal/auth/sms.go:6` - 使用 SHA1

**建议**: 评估是否可升级至 SHA256，某些 DDNS 协议可能要求 SHA1。
