## 二、高危问题详情

### 2.1 敏感信息泄露 🔴

#### 问题 1: 默认密码打印到控制台
**位置**: `internal/users/manager.go:183`
```go
fmt.Printf("   密码: %s\n", defaultPassword)
```
**风险**: 首次启动时默认密码输出到 stdout，可能被日志收集系统捕获
**建议**: 
- 改为写入临时文件，提示用户查看
- 或通过安全渠道（邮件）发送
- 至少确保不写入日志文件

#### 问题 2: TOTP Secret 解密失败泄露
**位置**: `internal/auth/enhanced_mfa_manager.go:137`
```go
fmt.Printf("警告：解密用户 %s 的 TOTP Secret 失败：%v\n", userID, err)
```
**风险**: 错误消息包含用户ID和解密失败详情，可能被用于推断系统信息
**建议**: 使用结构化日志，避免直接打印

#### 问题 3: TLS InsecureSkipVerify 配置
**位置**: 
- `internal/backup/cloud.go:136`
- `internal/backup/sync.go:670`
- `internal/cloudsync/providers.go:399`

```go
TLSClientConfig: &tls.Config{InsecureSkipVerify: true}
```
**风险**: 跳过 TLS 证书验证可能导致中间人攻击
**当前缓解**: 添加了 `#nosec G402` 注释，声称由用户配置控制
**建议**: 
- 添加配置警告日志
- 仅在明确确认风险的情况下允许启用

### 2.2 输入验证问题 🔴

#### 问题 4: 脚本执行黑名单不完整
**位置**: `internal/snapshot/executor.go:25-35`
```go
var dangerousCommands = []string{
    "rm -rf /",
    "rm -rf /*",
    "mkfs",
    // ... 仅 9 个危险命令
}
```
**风险**: 黑名单可被绕过，例如：
- `rm -rf /home`（不在黑名单中）
- `> /dev/sda` 的变体
- Base64 编码的恶意命令
**建议**: 
- 使用白名单机制替代黑名单
- 或使用沙箱环境执行脚本
- 添加命令审计日志

#### 问题 5: filepath.Walk 竞态条件 (TOCTOU)
**位置**: 
- `internal/backup/sync.go:369`
- `internal/snapshot/replication.go:821`
- `internal/backup/restore.go:180`

```go
err := filepath.Walk(dir, func(path string, info os.FileInfo, err error) error {
    // 在检查和使用之间存在时间窗口
})
```
**风险**: Time-of-Check-Time-of-Use 竞态，文件可能在检查后被修改
**建议**: 
- 使用 `fs.FS` 接口
- 或在操作前重新验证文件状态

## 四、低危问题详情

### 4.1 代码质量问题

1. **环境变量安全**: `NAS_CSRF_KEY` 未设置时有回退逻辑，但回退密钥安全性较低
2. **日志格式不一致**: 混用 `log.Printf` 和 `fmt.Printf`
3. **错误消息国际化**: 部分错误消息包含中文，可能影响日志解析
4. **recover 使用**: 仅 3 处使用 recover，覆盖不完整

### 4.2 建议改进

1. 敏感操作添加审计日志
2. 统一使用结构化日志 (zap/logrus)
3. 敏感配置项添加加密存储
4. 定期轮换 CSRF Key
