### 发现的安全问题

#### 1. 🔴 高危：整数溢出风险 (G115)
**位置**: `internal/quota/optimizer/optimizer.go`, `internal/monitor/metrics_collector.go` 等多处
**详情**: uint64 到 int/int64 转换可能导致溢出
**影响**: 可能导致逻辑错误或服务拒绝

#### 2. 🔴 高危：TLS 配置风险 (G402)
**位置**: 多处 HTTP 客户端配置
**详情**: `InsecureSkipVerify` 可能被设置为 true
**影响**: 中间人攻击风险

#### 3. 🔴 高危：命令注入风险 (G702)
**位置**: 需要详细审查命令执行相关代码
**详情**: 污点分析发现潜在的命令注入路径
**影响**: 可能导致任意命令执行

#### 4. 🟡 中危：路径遍历风险 (G304, G703)
**位置**: 文件操作相关代码
**详情**: 文件路径由用户输入或变量构造
**影响**: 可能访问预期外的文件

#### 5. 🟡 中危：权限过于宽松 (G301, G306)
**位置**: 文件/目录创建时权限设置
**详情**: 部分文件使用 0755/0644 权限
**影响**: 敏感文件可能被非授权用户访问

#### 6. 🟡 中危：错误未检查 (G104)
**位置**: 526 处未检查错误返回值
**详情**: 大量函数调用未检查返回的错误
**影响**: 可能导致静默失败

#### 7. 🟢 低危：硬编码凭证 (G101)
**位置**: 7 处潜在硬编码凭证
**详情**: 需要人工确认是否为真实凭证

### 敏感信息泄露风险检查

#### ✅ 良好实践
1. **敏感字段不序列化**: 使用 `json:"-"` 标签防止密码等敏感信息序列化
   - `internal/backup/cloud.go`: `SecretKey string json:"-"`
   - `internal/backup/sync.go`: `SecretKey`, `Password` 字段
   - `internal/backup/manager.go`: `RemotePassword string json:"-"`

2. **凭证加密存储**: `internal/backup/credentials.go` 实现了 AES-GCM 加密存储凭证

3. **.env 文件**: 不存在 .env 文件，只有 .env.example 模板

#### ⚠️ 需要关注
1. **默认密码打印**: `internal/users/manager.go:194` 打印默认密码到标准输出
   ```go
   fmt.Printf("   密码: %s\n", defaultPassword)
   ```
   **建议**: 仅在调试模式或开发环境显示，生产环境应移除

2. **环境变量使用**: 敏感信息通过环境变量传入，符合最佳实践
   - `NAS_CSRF_KEY`
   - `GITHUB_TOKEN`
