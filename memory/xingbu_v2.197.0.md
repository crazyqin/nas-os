## 二、高危问题详情

### 1. G115 - 整数溢出 (74个)

**风险**: uint64 -> int64 转换可能导致溢出

**主要位置**:
- `internal/quota/optimizer/optimizer.go`
- `internal/optimizer/optimizer.go`

**修复建议**: 
- 使用 `math/big` 或检查边界
- 添加溢出检查函数

### 2. G703 - 路径遍历 (48个) ⚠️

**风险**: 用户输入可能被用于构造恶意路径

**主要位置**:
- `plugins/filemanager-enhance/main.go:537`
- `internal/webdav/server.go:863-864`

**修复建议**:
- 使用 `filepath.Clean()` 清理路径
- 验证路径是否在允许的根目录内
- 使用 `filepath.Rel()` 检查相对路径

### 3. G702 - 命令注入 (10个) 🔴

**风险**: 用户输入可能被注入到系统命令

**主要位置**:
- `internal/vm/snapshot.go:290-335`

**修复建议**:
- 使用 `exec.Command()` 的参数列表形式
- 严格验证所有用户输入
- 避免字符串拼接构造命令

### 4. G101 - 硬编码凭证 (7个)

**风险**: 代码中可能包含密码或密钥

**主要位置**:
- `internal/office/types.go:581`
- `internal/cloudsync/providers.go:670,1320`

**修复建议**:
- 移除硬编码凭证
- 使用环境变量或配置文件
- 审查是否为示例或测试代码

### 5. G122 - 文件系统竞态 (7个)

**风险**: filepath.Walk/WalkDir 中存在 TOCTOU 竞态

**主要位置**:
- `internal/snapshot/replication.go:827`
- `internal/plugin/hotreload.go:348`
- `internal/files/manager.go:897`

**修复建议**:
- 使用 root-scoped API
- 考虑使用 `fs.FS` 接口

### 6. G402 - TLS 配置不安全 (3个)

**风险**: InsecureSkipVerify 可能被设为 true

**主要位置**:
- `internal/ldap/client.go:73,107`
- `internal/auth/ldap.go:153`

**修复建议**:
- 仅在明确需要时启用
- 添加配置警告日志
- 提供安全的默认配置
