# v2.35.0 CI/CD 优化总结

## 修改文件列表

### 1. `.github/workflows/ci-cd.yml`

**修改内容：**
- 添加顶层 `permissions` 配置，采用最小权限原则
  - `contents: read` - 读取代码
  - `security-events: write` - 上传安全扫描结果（CodeQL, Trivy, gosec）
  - `id-token: write` - OIDC token 用于 keyless 签名
  - `actions: read` - 读取 workflow 信息
  - `checks: write` - 写入检查结果
- 更新缓存版本号 `CACHE_VERSION: 'v3'`
- 为 `dependency-scan` job 添加 `security-events: write` 权限
- 为多平台构建添加 Go 构建缓存加速

**解决问题：**
- 修复 CI/CD 因权限不足导致的失败
- 安全扫描结果可以正确上传到 GitHub Security

### 5. `Dockerfile`

**修改内容：**
- 更新版本注释为 v2.35.0
- 优化健康检查命令：
  - 使用 `curl` 替代 `wget`（更可靠）
  - 添加 `/api/v1/metrics` 端点检测
  - 支持更完善的健康状态检测

**健康检查逻辑：**
```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --start-period=30s --retries=3 \
    CMD (test -f /var/run/nasd.pid || pgrep -x nasd > /dev/null) && \
         curl -sf http://localhost:8080/api/v1/health > /dev/null 2>&1 && \
         curl -sf http://localhost:8080/api/v1/metrics > /dev/null 2>&1 || \
         (echo "Health check failed: service unhealthy" && exit 1)
```

### 6. `scripts/healthcheck.sh` (新增)

**新增内容：**
- 完整的健康检查脚本
- 支持三种检查模式：

  - `live` - 存活检查（用于 Kubernetes livenessProbe）
- 检查进程状态、健康端点、指标端点

## 监控增强说明

### 健康检查端点

项目已有完善的健康检查实现（`internal/performance/health.go`）：
- `/api/v1/health` - 完整健康状态
- `/api/v1/health/checks` - 详细检查结果

### 指标导出端点

项目已有 Prometheus 指标导出（`internal/performance/monitor.go`）：
- `/metrics` - Prometheus 格式指标
- `/api/v1/metrics` - JSON 格式指标

### 支持的指标

- **系统指标**：CPU、内存、磁盘、网络
- **存储指标**：IOPS、延迟、吞吐量
- **性能指标**：请求计数、延迟分布、错误率
- **缓存指标**：命中率、缓存大小
- **业务指标**：文件操作、搜索、上传下载

## 权限配置最佳实践

采用最小权限原则，所有 workflow 在顶层定义权限：

```yaml
permissions:
  contents: read           # 读取代码
  security-events: write   # 上传安全扫描结果
  id-token: write          # OIDC token 用于 keyless 签名
  packages: write          # 推送镜像（仅 docker-publish）
  attestations: write      # 写入 attestations（仅 docker-publish）
```

## 缓存优化

- 使用 `actions/cache@v4` 进行 Go 构建缓存
- 缓存键包含 Go 文件哈希，确保缓存有效性
- 多级缓存恢复策略
