## 中文

### 概述

感谢您对 NAS-OS 项目的关注！我们重视每一位用户的反馈，本文档将帮助您有效地提交问题和建议。

### 反馈渠道

#### 1. Bug 报告 🐛

如果您发现了 Bug，请通过以下方式报告：

1. **GitHub Issues**（推荐）
   - 访问 [Issues 页面](https://github.com/crazyqin/nas-os/issues)
   - 点击 "New Issue"
   - 选择 "Bug Report" 模板
   - 填写详细信息

2. **Bug 报告内容**
   - 清晰的问题描述
   - 复现步骤（逐步说明）
   - 期望行为 vs 实际行为
   - 环境信息（操作系统、版本等）
   - 相关日志或截图

#### 2. 功能建议 💡

如果您有新功能的想法：

1. **GitHub Discussions**（讨论阶段）
   - 访问 [Discussions 页面](https://github.com/crazyqin/nas-os/discussions)
   - 在 "Ideas" 分类下发帖讨论
   
2. **Feature Request**（正式提案）
   - 在 Issues 页面选择 "Feature Request" 模板
   - 说明使用场景和预期效果

#### 3. 安全问题 🔒

如果您发现了安全漏洞：

- **请勿**在公开 Issues 中报告
- 发送邮件至：security@example.com
- 我们会在 48 小时内回复

#### 4. 文档问题 📖

发现文档错误或缺失：

- 在 Issues 中选择 "Documentation" 标签
- 说明具体问题和改进建议

## English

### Overview

Thank you for your interest in NAS-OS! We value every user's feedback. This document will help you submit issues and suggestions effectively.

### Feedback Channels

#### 1. Bug Reports 🐛

If you find a bug, please report it through:

1. **GitHub Issues** (Recommended)
   - Visit [Issues page](https://github.com/crazyqin/nas-os/issues)
   - Click "New Issue"
   - Select "Bug Report" template
   - Fill in details

2. **Bug Report Contents**
   - Clear problem description
   - Reproduction steps (step by step)
   - Expected behavior vs actual behavior
   - Environment info (OS, version, etc.)
   - Relevant logs or screenshots

#### 2. Feature Suggestions 💡

If you have ideas for new features:

1. **GitHub Discussions** (Discussion phase)
   - Visit [Discussions page](https://github.com/crazyqin/nas-os/discussions)
   - Post under "Ideas" category
   
2. **Feature Request** (Formal proposal)
   - Select "Feature Request" template in Issues
   - Explain use case and expected outcome

#### 3. Security Issues 🔒

If you discover a security vulnerability:

- **Do NOT** report in public Issues
- Email: security@example.com
- We will respond within 48 hours

#### 4. Documentation Issues 📖

Found documentation errors or missing content:

- Select "Documentation" label in Issues
- Specify the issue and improvement suggestions
