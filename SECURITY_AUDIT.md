## 2. 高危漏洞详情

### 2.1 命令注入 (G702) - 10 处

**位置**: `internal/vm/manager.go`, `internal/vm/snapshot.go`

**问题代码示例**:
```go
cmd := exec.CommandContext(ctx, "virsh", "-c", "qemu:///system", "start", vm.Name)
```

**风险等级**: 高危  
**当前状态**: 代码已添加 `#nosec` 注释，声明已通过 `validateConfig()` 验证输入

**修复建议**:
1. 确保 `validateConfig()` 严格限制 VM 名称字符集 (仅允许 `[a-zA-Z0-9_-]`)
2. 使用参数白名单验证而非黑名单
3. 添加单元测试覆盖边界情况
4. 考虑使用 `exec.Command` 的参数数组形式而非字符串拼接

### 2.2 路径遍历 (G703) - 49 处

**位置**: `server.go`, `main.go` (WebDAV 服务)

**问题代码示例**:
```go
fullPath := filepath.Join(rootPath, r.URL.Path)
os.WriteFile(fullPath, data, 0640)
```

**风险等级**: 高危  
**当前状态**: 部分代码已添加 `#nosec` 注释，声明通过 `resolvePath()` 验证

**修复建议**:
1. 实现 `resolvePath()` 函数强制检查路径规范化：
   ```go
   func resolvePath(root, userPath string) (string, error) {
       fullPath := filepath.Join(root, userPath)
       absPath, err := filepath.Abs(fullPath)
       if err != nil {
           return "", err
       }
       absRoot, _ := filepath.Abs(root)
       if !strings.HasPrefix(absPath, absRoot) {
           return "", errors.New("path traversal detected")
       }
       return absPath, nil
   }
   ```
2. 添加符号链接检查防止 TOCTOU 攻击
3. 对所有文件操作统一使用路径验证中间件

### 2.3 整数溢出 (G115) - 108 处

**位置**: 多个文件

**问题**: `uint64 -> int` 类型转换可能导致溢出

**修复建议**:
1. 使用 `int64` 替代 `int` 处理大数值
2. 添加边界检查：
   ```go
   if size > math.MaxInt64 {
       return errors.New("size overflow")
   }
   safeSize := int(size)
   ```

### 2.4 硬编码凭证 (G101) - 9 处

**修复建议**:
1. 审查所有硬编码凭证
2. 迁移到环境变量或密钥管理系统
3. 标记为 `#nosec G101` 仅当确认是示例/默认值

### 2.5 其他高危漏洞

## 9. 应用中心安全设计 (2026-03-28 新增)

### 9.1 安全隔离架构

**三层隔离模型**：

```
┌─────────────────────────────────────────────────────────────┐
│                     应用安全隔离架构                          │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  第一层：用户/权限隔离                                         │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  UID 隔离 (50000-60000)  │  非 root 运行  │  目录 quota │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                               │
│  第二层：网络隔离                                             │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  Bridge 网络  │  端口白名单  │  应用间通信隔离  │  iptables │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                               │
│  第三层：数据访问控制                                         │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  路径策略  │  只读/写入/完全访问  │  审批流程  │  审计日志 │  │
│  └───────────────────────────────────────────────────────┘  │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 9.2 权限隔离配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 应用 UID 范围 | 50000-60000 | 每个应用独立 UID |
| 禁止 root 运行 | true | 所有应用以非特权用户运行 |
| Capabilities | DROP ALL | 默认禁用所有特权 |
| Seccomp | default | 应用默认安全策略 |
| CPU 限制 | 50% | 默认 CPU 上限 |
| 内存限制 | 2GB | 默认内存上限 |

### 9.3 网络访问控制

**禁止访问端口**（NAS 管理端口）：
- 80/443 (NAS Web UI)
- 22 (SSH)
- 873 (rsync)
- 2049 (NFS)

**允许访问服务**：
- DNS (53)
- NTP (123)
- LDAP (389) - 可选

**应用间通信**：默认禁止，需白名单例外

### 9.4 数据访问策略

| 路径类型 | 默认策略 | 审批要求 |
|----------|----------|----------|
| 系统路径 | DENY | 不可访问 |
| 用户数据 | READONLY | 无需审批 |
| 媒体目录 | READONLY | 无需审批 |
| 共享目录 | APPROVAL | 需管理员审批 |
| 备份目录 | WRITEONLY | 无需审批 |

### 9.5 应用安全审计

**审计级别**：
| 级别 | 记录内容 |
|------|----------|
| minimal | 启动/停止/崩溃 |

| detailed | + 文件访问/网络访问 |
